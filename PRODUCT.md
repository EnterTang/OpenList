# Product

## Register

product

## Users
OpenList is used by people who manage, mount, browse, transfer, and share files across many cloud storage providers from a self-hosted web interface.

## Product Purpose
OpenList provides a unified file manager and API surface for heterogeneous storage backends. Success means users can configure storage and automation workflows with confidence, then let routine transfer and organization tasks run without manual babysitting.

## Brand Personality
Practical, resilient, and transparent. The interface should feel like a trustworthy operations tool rather than a promotional site.

## Anti-references
Avoid decorative dashboards, marketing-style hero layouts inside the admin app, unclear destructive actions, and custom controls that obscure standard file-management workflows.

## Design Principles
- Keep operational controls close to the workflow they affect.
- Make destructive automation explicit and easy to audit.
- Prefer dense, scannable configuration over decorative layout.
- Preserve familiar form, table, and navigation patterns.

## Accessibility & Inclusion
Use accessible contrast, visible focus states, clear labels, and reduced-motion friendly interactions. Configuration screens should remain usable in Chinese and English with long provider names and paths.
