# Mobile 139 ETF Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add focused Mobile 139 `.etf` generation, restore, playback, cleanup, and TMDB/category-path support for the `139Yun` `personal_new` driver.

**Architecture:** Add small internal packages for ETF metadata, media category matching, media recognition, and TMDB lookup, then wire them into `drivers/139`. Keep global TMDB/category configuration in existing OpenList settings and keep ETF behavior driver-opt-in.

**Tech Stack:** Go, existing OpenList setting/driver/fs/server packages, `gopkg.in/yaml.v3`, standard `net/http` test servers.

---

- Create `internal/etfmeta/etfmeta.go`: ETF encode/decode, file-name helpers, restore-name resolution.

- Create `internal/media/tmdb/tmdb.go`: TMDB client and candidate scorer.
- Create `internal/media/tmdb/tmdb_test.go`: fake-server search/detail/scoring tests.
- Modify `internal/conf/const.go`: add `TMDBApiKey`, `TMDBApiBaseURL`, `TMDBLanguage`, `MediaCategoryRules`.
- Modify `internal/bootstrap/data/setting.go`: add default global settings.

- Modify `drivers/139/meta.go`: add opt-in ETF configuration fields.
